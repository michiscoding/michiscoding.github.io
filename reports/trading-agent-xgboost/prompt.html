The current DQN approach trains one separate reinforcement learning agent per stock. Each agent learns BUY / SELL / HOLD decisions by repeatedly walking through the same historical price period. The issue is that even though 750 days × 50 episodes looks like ~37,500 training steps, the model is still only learning from ~750 unique market days per stock. That creates a high risk of overfitting, especially with only 3–4 years of daily data.

The preferred architecture is a global cross-sectional ranking model using something like XGBoost, LightGBM, or CatBoost.

Instead of training one model per ticker, we train one model across all tickers and dates. Each row should represent one stock on one date:

date | ticker | features | future_return_target
The model’s job is not to directly choose BUY / SELL / HOLD. Its job is to predict or rank which stocks are most likely to outperform over the next holding period, such as the next 5, 10, or 20 trading days.

Example target:

future_5_day_return = adjusted_close[t+5] / adjusted_close[t] - 1
Then each day, the system scores every stock, ranks them, and only buys the top-ranked stocks if the predicted return/edge is strong enough.

High-level flow:

1. Load historical OHLCV data for all tickers.
2. Build features for each ticker/date.
3. Create forward-return targets.
4. Train one global ranking/regression model across all ticker-date rows.
5. Use walk-forward validation, not random train/test splitting.
6. Each trading day, score all stocks.
7. Rank stocks by predicted forward return or risk-adjusted return.
8. Buy the top-ranked stocks only if they clear a threshold.
9. Apply portfolio/risk rules: max position size, volatility scaling, cash allocation, transaction costs, drawdown controls.
10. Compare performance against baselines.
Suggested features:

Price/return features:
- 1-day, 5-day, 10-day, 20-day returns
- moving average distances: price vs 10/20/50/200-day MA
- momentum indicators
- rolling volatility
- rolling max drawdown
- gap from previous close, if using open prices

Volume/liquidity features:
- volume change
- rolling average volume
- volume relative to 20-day average

Technical indicators:
- RSI
- MACD
- Bollinger Band position
- ATR / volatility indicators

Market context:
- S&P 500 / Nasdaq return
- sector ETF return
- VIX, if available
- interest rates / treasury yield data, if available

Cross-sectional features:
- stock return minus market return
- stock return minus sector return
- relative strength rank among all tickers
- volatility rank among all tickers
Suggested model target options:

Option A: Predict raw forward return
target = 5-day or 10-day future return

Option B: Predict risk-adjusted return
target = future return / recent volatility

Option C: Rank stocks by future return
Use ranking objective if using LightGBM ranker/XGBoost ranker
Recommended first implementation:

Model:
LightGBM or XGBoost regression/ranking model

Target:
5-day or 10-day forward return

Training format:
One combined dataset with all tickers

Validation:
Walk-forward validation

Trading rule:
Each day, buy top 1–3 ranked stocks only if predicted return > threshold.
Otherwise hold cash.

Risk:
Cap individual stock position size.
Include transaction costs/slippage.
Use volatility-adjusted sizing if possible.
Validation should be time-based. Do not randomly shuffle rows because that leaks future information.

Example split:

Train: 2014–2021
Validate: 2022–2023
Test: 2024–2025
Or use walk-forward validation:

Train on earlier period → test on next period
Expand training window → test next period

Important note on data:

Using 10 years of data would be better than 3–4 years. But the bigger improvement is not only more years — it is more tickers, more features, and better validation.

Current data scale:

9 stocks × ~750 days = ~6,750 unique stock-day rows
Better data scale:

100–500 stocks × 10 years × 252 trading days
= ~252,000 to 1,260,000 unique stock-day rows
Recommended data upgrade:

Minimum:
- 10 years of adjusted OHLCV data
- More tickers, ideally 100+
- Market index data
- Sector ETF/index data
- VIX or volatility regime data
- Transaction cost assumptions

Nice to have:
- Earnings dates
- EPS/revenue surprises
- Analyst revisions
- Fundamentals/valuation ratios
- News/sentiment data
- Options/implied volatility data
The main reason for this change is that DQN is a complex trial-and-error decision system, but with limited unique data it can easily overfit. A boosted-tree ranking model is less flashy, but it is more practical for this dataset because it learns across all stocks together and directly answers the better trading question:

“Out of all available stocks, which ones look best right now?”
Rather than:

“For this one stock, should I buy, sell, or hold today?”
So the implementation goal should be:

Build a global stock-ranking system that predicts forward returns, ranks all available stocks each day, trades only the strongest signals, and validates performance against simple baselines using walk-forward testing.
