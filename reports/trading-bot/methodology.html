How the Trading Agent Works

A plain-English breakdown of training, backtesting, and live deployment — and how each phase differs from the next.

The Big Picture: Three Phases

Everything we did falls into three distinct phases. The most important thing to understand is that the data in each phase is completely different — the model never "cheats" by seeing future prices during training.

Phase 1
Training

The agent learns from years of historical price data it's allowed to study. This is where the actual "learning" happens. Data: 2020–2023.

Phase 2
Backtesting

We run the trained model on 2025 prices it has never seen. No learning happens here — we're just measuring whether it actually works. Data: 2025.

Phase 3
Live Paper Trading

The model is deployed. It gets real prices daily, places real (fake-money) orders, and retrains slightly each day from its own live results. Data: 2026 onward.

Phase 1 — Training

What is the agent actually learning?

The agent is a neural network that answers one question every day: given the last 10 days of price movements, should I BUY, SELL, or HOLD?

It learns by trial and error over thousands of simulated trading days. When it makes a profitable trade, it gets a positive reward. When it loses money, it gets a negative reward. Over time it updates its internal weights to make better decisions.

Analogy: Imagine playing a video game where you can only press three buttons (BUY, SELL, HOLD) and your score is your profit. After enough attempts, you start recognizing which patterns of price movement tend to be good signals. The agent does the same thing — but with math instead of intuition.

The algorithm: Deep Q-Learning (DQN)

The specific technique is called Deep Q-Learning. "Deep" means we use a neural network. "Q" refers to Q-values — a number representing "how good is it to take action X in situation Y." The agent learns Q-values for all three actions (BUY/SELL/HOLD) across all market situations, then always picks the highest one.

Two key tricks that make it actually work:

  • Experience Replay: Instead of learning from each trade immediately (which causes instability), the agent stores its experiences in a memory bank and randomly samples from them later. This breaks correlations in the data and makes training more stable.
  • Double DQN: Uses two copies of the network — one that makes decisions, one that evaluates them. This prevents the agent from systematically overestimating how good its trades are.

What counts as one "training run"?

  • 1

    Load 3–4 years of daily closing prices for one ticker (e.g., GOOGL 2020–2023 = ~750 trading days).

  • 2

    Run the agent through all 750 days, making a decision each day. This is called one episode.

  • 3

    After enough experiences accumulate (≥32), sample a random batch and run gradient descent to update the network weights.

  • 4

    Repeat for 50 episodes — 50 full passes through all 750 days. That's 37,500 decision points total, each one a learning opportunity.

  • 5

    Save the trained model to disk. Done.

Why did it take so long?

We trained 9 separate models — one per stock. Each ran 50 episodes through ~750 days of data, with a neural network update after each batch. TensorFlow (the deep learning framework) is doing thousands of matrix multiplications per second, but it still adds up. Longer training = more stable, well-converged weights. We also had to debug a platform issue (TensorFlow's x86 binaries used CPU instructions that don't work under Apple Silicon's Rosetta emulation), which cost time before we switched to the Apple Silicon-native build.

Phase 2 — Backtesting (the "test set")

Once training is done, the model's weights are frozen — it can't learn anymore in this phase. We then run it against 2025 price data it has never seen, and record every BUY/SELL/HOLD decision it makes.

This is the most important evaluation. Training performance is meaningless — any model can memorize its training data. What matters is whether the patterns it learned actually generalize to new market conditions.

Why 2025 specifically? We trained on 2020–2023. We skipped 2024 entirely (keeping it as a buffer to reduce look-ahead bias), then tested on 2025 — a full year of data that postdates everything the model ever touched. This is sometimes called "walk-forward" analysis.

Results summary

8 out of 9 models were profitable on 2025 data they'd never seen. The GOOGL Modern model (trained on 2020–2023 Google prices) made +$5,977 over the year — 49 round trips at an average of $122 profit each. See results.html for the full breakdown with charts.

Phase 3 — Live Paper Trading (deployed)

This is where we are now. The model is deployed — running in production, just with paper money instead of real money (Alpaca's paper trading environment uses real market prices but simulated funds).

What happens every weekday at 4:15 PM ET

  • 1

    Fetch the last 10+ days of real closing prices from Alpaca's data API.

  • 2

    Build the current market "state" — same format the model was trained on (10-day sigmoid-normalized price differences).

  • 3

    The model outputs a decision: BUY, SELL, or HOLD.

  • 4

    If BUY: place a $10,000 market order (fractional shares) on Alpaca paper account.

  • 5

    Online learning: feed yesterday's outcome back in as a training experience. If we bought and price went up, that's a positive reward. If price dropped, negative. The model retrains slightly from this — adapting to 2026 market conditions in real time.

  • 6

    Log everything: action, price, P&L, running total.

How is this different from the backtest?

The backtest was a replay — we already had all of 2025's prices, so we could run through the whole year at once and calculate total profit. Live trading is real-time: the model only ever sees prices up to today, and actually places orders that fill at real bid/ask spreads. It also learns from each live trade, so the model you're running in month 2 is slightly different from month 1. The backtest model was static; the live model evolves.

The key experiment baked into this setup

Two models trade the same stock (GOOGL) — one trained on recent data (2020–2023), one on decade-old data (2010–2018). In the 2025 backtest, the modern model outperformed the baseline by 12x on identical test data. The live paper trading will tell us if that gap holds in real-time 2026 conditions, or if the online learning loop closes it.

What We're Watching For

The paper trading period (targeting ~30 days) is meant to answer three questions:

  • Does the backtest ranking hold? GOOGL Modern was #1 in 2025 data. Is it still #1 on live 2026 data?
  • Does online learning help? The model adapts daily. Does it improve over the first 30 days, or does it drift?
  • Is the system ready for real money? If live results track the backtest directionally, we go live with real capital.
