d365 to/from sftp file transfer

flows — april 8 2026

what it does

this solution includes three workflows to automate importing and exporting bank files. the outbound workflow intakes files sharepoint that were dropped in from d365, encrypts its contents, then uploads them to the bank's sftp server. the inbound workflow intakes the file from the designated sftp folder and creates the file in sharepoint. the last workflow is for notifications and error handling. if a file fails to process, it moves to the 'error' sharepoint folder, and sends an email to notify the team. otherwise, it moves to the 'processed' folder.

how it works

this solution was created with an azure resource group containing a logic app, function app, and a key vault.
